批量刷盘失败会永久丢失数据

internal/repository/batch.go:75-82
○ 待处理

Flush 先把缓冲区摘出 batch := b.buf 再写库，flushBatch 出错时这批行已脱离缓冲区，调用方仅 log 不补救，导致永久丢失。

修复：flushBatch 失败时把 batch 重新 prepend 回 b.buf 以便下次重试。

✓ 已实现：flushBatch 出错时执行 b.buf = append(batch, b.buf...) 把行放回缓冲区，下次刷盘重试。

修复：用 DSN ?_pragma=... 参数让所有连接一致（modernc 支持）。
